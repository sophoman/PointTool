一、软件介绍
本软件主要用于中控平台OTS开发中DCS位号与模型位号通信点表的生成。
二、Excel格式介绍
Excel主要有三个部分需要注意。
1.Sheet名
Sheet的名字决定了这个工作表的生成的数据属于哪一个工作站。当我们第一次创建。
比如我在通信软件中创建了一个服务器名为FCS1902，那么我的Sheet名也应该为FCS1902这样生成的CSV文件的目标节点才会是正确的内容。
2.表头
Excel的表头主要由六个内容组成：
目标节点位号：DCS中对应的位号名。
源节点位号：OTS模型对象的位号名。
Suffix：OTS模型中想要使用位号的后缀。
组：在通讯文件中分配到那个组别，不同的组对应不同的数据流向。
描述：对该通信位号的（主要是DCS位号）的具体作用的描述。
公式：在通信过程中，如果需要对通信数据进行处理，需要使用到公式，默认为空。
每个Excel必须将首行空出来作为表头行，其中的内容可以不是以上，但是还是表头必须得用相应的内容，不然会造成生成失败的问题。
对于组的选择，主要有以下几种：
AI：OTS->DCS
AO：DCS->OTS
DI：OTS->DCS
DO：DCS->OTS
以上四个组主要用于DCS中自带控制站中的位号。如果在自带控制站中添加自己的位号，放入上面四个组也可以。
FOV：DCS->OTS   主要用于现场阀门VI VX
PFOV: DCS->OTS   主要用于泵的前后阀门
PSV:OTS->DCS     主要用于安全阀反馈
Spare_AI:OTS->DCS 主要用于主阀_A
CV:DCS->OTS      主要用于组阀_I,_O,_B
RD:DCS->OTS      自定义
WR:OTS->DCS     自定义
以上组别主要用于现场的相关的位号。当然如果不想使用上面的一些组，也可以自定义，在后续详细配置中会提到。
公式在对点中也是一个经常会使用到的内容。比如DCS实际输出的信号为ON，但是在模型中实际正确的信号为OFF，这时可是使用公式not(x)，对DCS传入模型的信号取反，这样模型中实际接收到信号就为OFF。
常用的公式主要有:not(x)，abs(x)，x+NUM，x-NUM，x*NUM，x/NUM，其中NUM为一个具体的数。
3.内容
Excel的内容主要正如上面所说，主要由六个内容组成，其中目标节点位号、源节点位号、Suffix、组为必填内容，描述、公式为选填内容。当Excel准备完毕后，即可进行生成，生成的默认方式为1对1生成，即每一行Excel中的点，在生成的CSV中也只占一行。

三、主面板
文件路径：Excel的文件路径，可通过浏览进行选择。
硬连接/软连接：硬连接和软连接的切换，暂时只支持硬连接
生成路径：要生成的CSV文件的保存位置，可通过浏览进行选择。
设为默认：激活后下次再次打开时，会自动填充文件路径和生成路径。
>>		：打开详细配置面板。
当一切准备完毕后，点击生成按钮，会生成三个文件:CSV文件、NA文本文件、ND文本文件。NA文本中主要记录了需要导入到DCS点表中的自定义模拟量位号，ND文件中主要记录需要导入到DCS点表中的自定义开关量位号。
如果想要在Excel文件打开的时候也可以进行CSV文件的生成，需要在Excel中打开共享模式。
注意：启用共享模式后，不能对Sheet进行删除。
四、规则配置面板
规则配置面板主要用于对一些特定的需求进行自定义配置，比如泵的现场点的生成、显示具体组分含量等需求。
图4.1 规则配置界面
其中默认包含了几个常用的配置，可在下拉菜单中查看。
标记：标记是该软件识别Excel哪些行需要使用特定规则的一种标识，一般来说，一张表对应一个标记。每张表的标记必须是不同的。
添加表：添加一张新的空表，会要求输入一个名字和特定标记。添加后可在下拉菜单查看。
删除表：删除选中的一张表，默认的几张表无法删除。
重置：将整个规则配置重置为初始状态，即只含有默认规则。
修改：修改选中表的标记。
添加行：向选中规则中添加一行。
删除行：删除选中的行，第一列为灰色的行无法删除。
另存：将规则另存为压缩文件。
载入：加载已经另存的压缩文件。
保存：保存修改的规则至内存中。
规则创建好以后，可以像二、3中泵的对点的使用方式使用这些规则。
